GET /api/status

GET /api/status

List all projects and their current documentation generation status.

Overview

The GET /api/status endpoint returns a summary of every project registered in docsfy along with its current generation state. Use this endpoint to monitor ongoing generation jobs, verify completed builds, or detect failures across all your documentation projects at a glance.

Request

GET /api/status

No query parameters, request body, or authentication are required.

Example Request

curl http://localhost:8000/api/status

import httpx

response = httpx.get("http://localhost:8000/api/status")
projects = response.json()

Response

Returns a JSON array of project objects. Each object contains the project name, its source repository, and the current generation status.

Response Fields

Field Type Description
name string Unique project identifier derived from the repository name
repo_url string GitHub repository URL used for generation
status string Current generation status: generating, ready, or error
last_generated string \| null ISO 8601 timestamp of the last successful generation, or null if never completed
last_commit_sha string \| null Git commit SHA from the most recent generation run

Status Values

Status Description
generating The documentation pipeline is actively running. The project is being cloned, planned, or rendered.
ready Generation completed successfully. The static HTML site is available to serve or download.
error Generation failed at some stage. Check project details for logs.

[]

Status Lifecycle

A project's status follows this lifecycle through the four-stage generation pipeline:

POST /api/generate
       │
       v
  ┌────────────┐     ┌─────────────────────────────────┐
  │ generating  │────>│  Clone → Plan → Content → HTML  │
  └────────────┘     └──────────────┬──────────────────┘
                                    │
                       ┌────────────┴────────────┐
                       v                         v
                 ┌──────────┐             ┌──────────┐
                 │  ready   │             │  error   │
                 └──────────┘             └──────────┘

  1. generating — Set immediately when a POST /api/generate request is accepted. The project stays in this state while the pipeline progresses through cloning, AI planning, AI content generation, and HTML rendering.

  2. ready — Set when the HTML renderer completes successfully and the static site is written to /data/projects/{name}/site/. The docs are now available at GET /docs/{name}/ and can be downloaded via GET /api/projects/{name}/download.

  3. error — Set if any pipeline stage fails. Use GET /api/projects/{name} to retrieve detailed error logs and determine which stage caused the failure.

Note: Re-running POST /api/generate for a project in error or ready status resets it back to generating and starts a fresh pipeline run.

Storage

Project status metadata is persisted in the SQLite database at /data/docsfy.db. This ensures status information survives service restarts.

/data/docsfy.db          # SQLite: project name, repo URL, status,
                         #         last_generated, last_commit_sha,
                         #         generation history, logs

The filesystem stores the generated artifacts separately:

/data/projects/{name}/
  ├── plan.json          # Documentation structure from AI planner
  ├── cache/
  │   └── pages/*.md     # AI-generated markdown (cached)
  └── site/              # Final rendered HTML (available when status is "ready")

Usage Patterns

Polling for Generation Completion

After triggering a generation with POST /api/generate, poll /api/status to check when the project finishes:

import httpx
import time

def wait_for_project(base_url: str, project_name: str, timeout: int = 600):
    """Poll until a project leaves the 'generating' state."""
    deadline = time.time() + timeout
    while time.time() < deadline:
        response = httpx.get(f"{base_url}/api/status")
        response.raise_for_status()
        for project in response.json():
            if project["name"] == project_name:
                if project["status"] != "generating":
                    return project
        time.sleep(5)
    raise TimeoutError(f"{project_name} still generating after {timeout}s")

result = wait_for_project("http://localhost:8000", "my-api")
print(f"Status: {result['status']}")

Tip: For details on a specific project — including page listings, generation logs, and commit metadata — use GET /api/projects/{name} instead.

Filtering by Status

The endpoint returns all projects. Filter client-side to find projects in a specific state:

import httpx

response = httpx.get("http://localhost:8000/api/status")
projects = response.json()

errored = [p for p in projects if p["status"] == "error"]
if errored:
    print(f"{len(errored)} project(s) failed generation:")
    for p in errored:
        print(f"  - {p['name']} ({p['repo_url']})")

Health Monitoring with jq

# Count projects by status
curl -s http://localhost:8000/api/status | \
  jq 'group_by(.status) | map({status: .[0].status, count: length})'

# List only projects currently generating
curl -s http://localhost:8000/api/status | \
  jq '[.[] | select(.status == "generating")]'

# Get names of all failed projects
curl -s http://localhost:8000/api/status | \
  jq -r '[.[] | select(.status == "error") | .name] | .[]'

Related Endpoints

Endpoint Description
POST /api/generate Start documentation generation for a repository
GET /api/projects/{name} Get detailed project info including pages and logs
DELETE /api/projects/{name} Remove a project and its generated docs
GET /api/projects/{name}/download Download the generated site as a .tar.gz archive
GET /docs/{project}/{path} Serve the generated static HTML documentation

Warning: The /api/status endpoint reflects the live state of the database. If a generation process is forcefully interrupted (e.g., the container is restarted), a project may remain stuck in generating status. Re-submit the project via POST /api/generate to restart the pipeline.
