AI Providers

AI Providers

docsfy supports three AI CLI tools for generating documentation from your repositories. Each provider analyzes your codebase, plans the documentation structure, and writes the content. You can switch between providers at any time through environment configuration.

Supported Providers

Provider CLI Binary Best For
Claude Code claude Default provider; strong code analysis and structured output
Gemini CLI gemini Alternative provider with Google AI models
Cursor Agent agent Workspace-aware provider from the Cursor editor ecosystem

Configuration

All provider settings are controlled through environment variables. Set them in your .env file or pass them directly to the container.

Core Settings

# AI Configuration
AI_PROVIDER=claude          # Provider name: claude, gemini, or cursor
AI_MODEL=claude-opus-4-6[1m] # Model identifier (provider-specific)
AI_CLI_TIMEOUT=60           # Timeout in minutes for AI CLI operations
Variable Default Description
AI_PROVIDER claude Which AI provider to use
AI_MODEL claude-opus-4-6[1m] The model identifier passed to the CLI tool
AI_CLI_TIMEOUT 60 Maximum time (in minutes) for a single AI invocation

Tip: The timeout applies per AI invocation, not per page. Large repositories with many pages may require a higher timeout value since each page in the documentation plan triggers a separate AI call.

Switching Providers

To switch providers, update AI_PROVIDER and AI_MODEL in your .env file:

# Switch to Gemini
AI_PROVIDER=gemini
AI_MODEL=gemini-2.5-pro

# Switch to Cursor Agent
AI_PROVIDER=cursor
AI_MODEL=claude-opus-4-6

Then restart the service for the change to take effect.

Note: Each provider has its own authentication requirements. Make sure the appropriate credentials are configured before switching. See the provider-specific sections below.

Claude Code

Claude Code is the default provider. It uses the claude CLI binary and supports two authentication methods.

Command Format

claude --model <model> --dangerously-skip-permissions -p

The --dangerously-skip-permissions flag is required because docsfy runs non-interactively in a server context where permission prompts cannot be answered. The -p flag enables prompt mode, accepting input via stdin.

Authentication

Option 1: API Key

The simplest method. Set a single environment variable:

ANTHROPIC_API_KEY=sk-ant-...

Option 2: Google Cloud Vertex AI

For organizations using Google Cloud, Claude Code can authenticate through Vertex AI:

CLAUDE_CODE_USE_VERTEX=1
CLOUD_ML_REGION=us-east5
ANTHROPIC_VERTEX_PROJECT_ID=your-gcp-project-id

When using Vertex AI, mount your Google Cloud credentials into the container:

# docker-compose.yaml
services:
  docsfy:
    volumes:
      - ~/.config/gcloud:/home/appuser/.config/gcloud:ro

Warning: The gcloud volume is mounted read-only (:ro). Do not remove the :ro flag, as the container should never modify your host credentials.

Installation

Claude Code is installed automatically in the Docker image:

curl -fsSL https://claude.ai/install.sh | bash

Example Configuration

AI_PROVIDER=claude
AI_MODEL=claude-opus-4-6[1m]
ANTHROPIC_API_KEY=sk-ant-your-key-here

Gemini CLI

Gemini CLI uses Google's AI models and runs through the gemini binary.

Command Format

gemini --model <model> --yolo

The --yolo flag disables interactive confirmation prompts, allowing the CLI to run autonomously in the server environment.

Authentication

Set your Gemini API key:

GEMINI_API_KEY=your-gemini-api-key

Installation

Gemini CLI is installed via npm in the Docker image:

npm install -g @google/gemini-cli

Note: The Docker image includes Node.js and npm as system dependencies specifically for Gemini CLI installation.

Example Configuration

AI_PROVIDER=gemini
AI_MODEL=gemini-2.5-pro
GEMINI_API_KEY=your-gemini-api-key

Cursor Agent

Cursor Agent uses the agent CLI binary and differs from the other providers in how it handles the working directory.

Command Format

agent --force --model <model> --print --workspace <path>
Flag Purpose
--force Skip interactive prompts
--model Specify the AI model
--print Output results to stdout instead of applying changes
--workspace Point to the repository path for code exploration

Working Directory Handling

Unlike Claude Code and Gemini CLI — which use the subprocess cwd parameter to set their working directory to the cloned repo — Cursor Agent manages its own working directory through the --workspace flag. This distinction is reflected internally via the uses_own_cwd configuration flag:

@dataclass(frozen=True)
class ProviderConfig:
    binary: str
    build_cmd: Callable
    uses_own_cwd: bool = False  # True only for Cursor Agent

Authentication

Set your Cursor API key:

CURSOR_API_KEY=your-cursor-api-key

When running in Docker, mount the Cursor configuration directory:

# docker-compose.yaml
services:
  docsfy:
    volumes:
      - ./cursor:/home/appuser/.config/cursor

Installation

Cursor Agent is installed automatically in the Docker image:

curl -fsSL https://cursor.com/install | bash

Example Configuration

AI_PROVIDER=cursor
AI_MODEL=claude-opus-4-6
CURSOR_API_KEY=your-cursor-api-key

How Providers Are Invoked

Regardless of which provider you choose, docsfy follows the same execution pattern for all AI operations.

Availability Check

Before starting a generation job, docsfy verifies that the configured provider is available by sending a lightweight "Hi" prompt. This catches misconfiguration — missing binaries, invalid credentials, or connectivity issues — before committing to a long-running generation pipeline.

Prompt Execution

All providers receive prompts through stdin and return output through stdout:

subprocess.run(cmd, input=prompt, capture_output=True, text=True)

AI invocations are wrapped in asyncio.to_thread() so they run without blocking the FastAPI event loop. Each call returns a tuple[bool, str] indicating success and the raw output.

Response Parsing

AI CLI output can vary in structure. docsfy uses a multi-strategy JSON extraction pipeline to reliably parse responses:

  1. Direct parse — attempt json.loads() on the raw output
  2. Brace matching — locate the outermost {...} JSON object
  3. Code block extraction — find JSON inside markdown ```json ``` blocks
  4. Regex fallback — recover partial JSON with pattern matching

This cascading approach handles differences in how each provider formats its output.

Docker Compose Reference

A complete docker-compose.yaml with all provider volumes and environment configuration:

services:
  docsfy:
    build: .
    ports:
      - "8000:8000"
    env_file: .env
    volumes:
      - ./data:/data
      # Claude Code - Vertex AI credentials (if using Vertex auth)
      - ~/.config/gcloud:/home/appuser/.config/gcloud:ro
      # Cursor Agent - configuration directory
      - ./cursor:/home/appuser/.config/cursor
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

Complete .env Reference

# --- Provider Selection ---
AI_PROVIDER=claude                    # claude | gemini | cursor
AI_MODEL=claude-opus-4-6[1m]         # Model identifier for the chosen provider
AI_CLI_TIMEOUT=60                    # Timeout per AI call in minutes

# --- Claude Code Authentication ---
# Option 1: API Key
# ANTHROPIC_API_KEY=sk-ant-...

# Option 2: Vertex AI
# CLAUDE_CODE_USE_VERTEX=1
# CLOUD_ML_REGION=us-east5
# ANTHROPIC_VERTEX_PROJECT_ID=your-project

# --- Gemini CLI Authentication ---
# GEMINI_API_KEY=your-gemini-key

# --- Cursor Agent Authentication ---
# CURSOR_API_KEY=your-cursor-key

# --- General ---
LOG_LEVEL=INFO

Tip: Only configure credentials for the provider you are actively using. Unused provider keys are ignored and can be left commented out.
