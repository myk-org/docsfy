Installation

Installation

docsfy can be installed using Docker (recommended), Docker Compose, or directly from source for local development.

Prerequisites

Before installing docsfy, ensure you have the following:

  • Git — for cloning repositories during documentation generation
  • An API key or credentials for at least one supported AI provider:
  • Claude Code (default) — requires ANTHROPIC_API_KEY or Vertex AI credentials
  • Gemini CLI — requires GEMINI_API_KEY
  • Cursor Agent — requires CURSOR_API_KEY

Docker (Recommended)

Docker is the recommended installation method. The image includes all system dependencies, AI CLI tools, and a non-root user configured for security.

Build the Image

git clone https://github.com/your-org/docsfy.git
cd docsfy
docker build -t docsfy .

The multi-stage Dockerfile uses python:3.12-slim as the base image and installs:

  • System dependencies: bash, git, curl, nodejs, npm, ca-certificates
  • Python package manager: uv
  • AI CLI tools: Claude Code, Cursor Agent, and Gemini CLI

Run the Container

docker run -d \
  --name docsfy \
  -p 8000:8000 \
  -v ./data:/data \
  --env-file .env \
  docsfy

This maps port 8000 and mounts a local ./data directory for persistent storage of the SQLite database and generated documentation sites.

Verify the Installation

curl http://localhost:8000/health

A healthy instance returns a success response on the /health endpoint.

Docker Compose

Docker Compose is the easiest way to run docsfy with all required volumes and configuration.

1. Create a .env File

Copy the environment template and configure your AI provider credentials:

cp .env.example .env

Edit .env with your settings:

# AI Configuration
AI_PROVIDER=claude
AI_MODEL=claude-opus-4-6[1m]
AI_CLI_TIMEOUT=60

# Claude - Option 1: API Key
ANTHROPIC_API_KEY=sk-ant-...

# Claude - Option 2: Vertex AI
# CLAUDE_CODE_USE_VERTEX=1
# CLOUD_ML_REGION=us-central1
# ANTHROPIC_VERTEX_PROJECT_ID=my-gcp-project

# Gemini
# GEMINI_API_KEY=

# Cursor
# CURSOR_API_KEY=

# Logging
LOG_LEVEL=INFO

2. Start the Service

docker compose up -d

This uses the following docker-compose.yaml configuration:

services:
  docsfy:
    build: .
    ports:
      - "8000:8000"
    env_file: .env
    volumes:
      - ./data:/data
      - ~/.config/gcloud:/home/appuser/.config/gcloud:ro
      - ./cursor:/home/appuser/.config/cursor
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

Volume Mounts

Mount Purpose
./data:/data Persistent storage for the SQLite database (docsfy.db) and generated project sites
~/.config/gcloud:/home/appuser/.config/gcloud:ro GCP credentials for Claude via Vertex AI (read-only)
./cursor:/home/appuser/.config/cursor Cursor configuration directory

Tip: If you are using Claude with an API key (ANTHROPIC_API_KEY) rather than Vertex AI, you can omit the gcloud volume mount. Similarly, omit the cursor mount if you are not using the Cursor provider.

3. Check Service Health

docker compose ps

The health check polls http://localhost:8000/health every 30 seconds and marks the container as unhealthy after 3 consecutive failures.

Local Development Setup

For contributing to docsfy or running it outside of Docker, you can set up a local development environment.

Requirements

Requirement Version
Python 3.12+
uv Latest
Node.js + npm For Gemini CLI (if used)
Git Any recent version

Warning: docsfy uses uv exclusively as its package manager. Do not use pip to install dependencies — it is not supported.

1. Install uv

If you don't have uv installed:

curl -LsSf https://astral.sh/uv/install.sh | sh

2. Clone and Install

git clone https://github.com/your-org/docsfy.git
cd docsfy
uv sync

uv sync installs all dependencies defined in pyproject.toml into a virtual environment managed by uv.

3. Install AI CLI Tools

You need at least one AI CLI tool installed locally. Install the one matching your chosen AI_PROVIDER:

Claude Code (default):

curl -fsSL https://claude.ai/install.sh | bash

Gemini CLI:

npm install -g @google/gemini-cli

Cursor Agent:

curl -fsSL https://cursor.com/install | bash

4. Configure Environment Variables

cp .env.example .env

Edit .env to set your AI provider and credentials. At minimum, set:

AI_PROVIDER=claude
ANTHROPIC_API_KEY=sk-ant-...

5. Create the Data Directory

mkdir -p data

docsfy stores all project data under /data by default:

/data/
  docsfy.db                # SQLite metadata database
  projects/
    {project-name}/
      plan.json            # Documentation structure from AI
      cache/
        pages/*.md         # AI-generated markdown (cached)
      site/                # Final rendered HTML site
        index.html
        *.html
        assets/
          style.css
          search.js
          theme-toggle.js
          highlight.js
        search-index.json

6. Run the Server

uv run uvicorn docsfy.main:app --host 0.0.0.0 --port 8000

The server starts on http://localhost:8000. For development with auto-reload:

uv run uvicorn docsfy.main:app --host 0.0.0.0 --port 8000 --reload

7. Verify

curl http://localhost:8000/health

Development Tooling

docsfy uses several tools for code quality. Install the development dependencies and hooks:

uv sync --all-extras
pre-commit install
Tool Purpose
Pre-commit Runs ruff (lint + format), mypy (strict), flake8, gitleaks, and detect-secrets on each commit
Tox Runs unused-code checks and unit tests via uv
Hatchling Python build system

Run the linters manually:

pre-commit run --all-files

Run the test suite:

tox

Environment Variable Reference

Variable Default Description
AI_PROVIDER claude AI provider to use: claude, gemini, or cursor
AI_MODEL claude-opus-4-6[1m] Model identifier passed to the AI CLI
AI_CLI_TIMEOUT 60 Maximum time in minutes for AI CLI operations
ANTHROPIC_API_KEY — API key for Claude (Option 1)
CLAUDE_CODE_USE_VERTEX — Set to 1 to use Claude via Vertex AI (Option 2)
CLOUD_ML_REGION — GCP region for Vertex AI
ANTHROPIC_VERTEX_PROJECT_ID — GCP project ID for Vertex AI
GEMINI_API_KEY — API key for Gemini
CURSOR_API_KEY — API key for Cursor
LOG_LEVEL INFO Logging verbosity: DEBUG, INFO, WARNING, ERROR

Authentication Setup by Provider

Claude with API Key

The simplest configuration. Set AI_PROVIDER=claude and provide your API key:

AI_PROVIDER=claude
ANTHROPIC_API_KEY=sk-ant-...

Claude with Vertex AI

For organizations using Google Cloud, Claude can be accessed through Vertex AI:

AI_PROVIDER=claude
CLAUDE_CODE_USE_VERTEX=1
CLOUD_ML_REGION=us-central1
ANTHROPIC_VERTEX_PROJECT_ID=my-gcp-project

When running with Docker Compose, mount your GCP credentials as a read-only volume:

volumes:
  - ~/.config/gcloud:/home/appuser/.config/gcloud:ro

Note: Ensure you have authenticated with gcloud auth application-default login on the host machine before starting the container.

Gemini

AI_PROVIDER=gemini
GEMINI_API_KEY=your-gemini-key

Cursor

AI_PROVIDER=cursor
CURSOR_API_KEY=your-cursor-key

When running with Docker Compose, mount the Cursor configuration directory:

volumes:
  - ./cursor:/home/appuser/.config/cursor

Container Security

The Docker image follows security best practices:

  • Runs as a non-root user (appuser)
  • Compatible with OpenShift (GID 0)
  • AI CLI tools are installed at build time
  • Health check endpoint available at /health

Warning: The container requires network access to clone Git repositories and communicate with AI provider APIs. Ensure your firewall and network policies allow outbound HTTPS traffic.
