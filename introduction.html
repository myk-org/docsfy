Introduction

Introduction

docsfy is an open-source FastAPI service that generates polished, Mintlify-quality static HTML documentation sites from GitHub repositories using AI CLI tools. Point it at any repository — public or private — and docsfy clones it, sends an AI agent to analyze the entire codebase, and produces a fully rendered documentation site with navigation, search, syntax highlighting, and dark/light theme support.

The Problem

Most software projects lack well-maintained, professionally formatted documentation. Writing docs is time-consuming, keeping them in sync with code changes is tedious, and producing a polished documentation site requires front-end skills that many backend developers don't prioritize. The result: outdated READMEs, scattered wiki pages, or no documentation at all.

The Solution

docsfy automates the entire documentation lifecycle. It leverages AI CLI tools — Claude Code, Cursor Agent, or Gemini CLI — to read and understand your codebase, then generates structured, accurate documentation rendered as a static HTML site. When your code changes, docsfy detects the diff and regenerates only the affected pages.

POST /api/generate  <-- repo URL

  +----------+   +--------------+   +------------+   +----------+
  |  Clone   |-->|  AI Planner  |-->| AI Content |-->|   HTML   |
  |  Repo    |   |  (plan.json) |   | Generator  |   | Renderer |
  +----------+   +--------------+   +------------+   +----------+

Key Features

  • AI-Powered Generation — Uses AI CLI tools with full repository access to produce documentation that understands your code, not just its surface structure
  • Multiple AI Providers — Supports Claude Code, Cursor Agent, and Gemini CLI through a unified provider interface
  • Incremental Updates — Tracks commit SHAs and regenerates only pages affected by code changes
  • Static Site Output — Generates self-contained HTML with sidebar navigation, client-side search (lunr.js), code syntax highlighting (highlight.js), and responsive design
  • Flexible Hosting — Serve docs directly from the docsfy API or download a .tar.gz archive to host anywhere
  • Dark/Light Theme — Built-in theme toggle with bundled CSS/JS assets

How It Works

docsfy runs a four-stage pipeline for each documentation generation request:

Stage 1: Clone

The target repository is shallow-cloned (--depth 1) to a temporary directory. Both SSH and HTTPS URLs are supported, and system git credentials handle private repository access.

Stage 2: Plan

An AI CLI tool runs with its working directory set to the cloned repository. The AI analyzes the entire codebase — its structure, source files, configuration, and existing documentation — and produces a plan.json that defines the pages, sections, and navigation hierarchy for the documentation site.

Stage 3: Generate

For each page defined in plan.json, the AI CLI tool runs again with full repository access, exploring the codebase as needed to write accurate content. Pages are generated concurrently using async execution with semaphore-limited concurrency. The output is cached as Markdown files:

/data/projects/{project-name}/
  plan.json             # doc structure from AI
  cache/
    pages/*.md          # AI-generated markdown (cached for incremental updates)

Stage 4: Render

Markdown pages and the plan are converted into polished static HTML using Jinja2 templates with bundled CSS/JS assets. The final site includes sidebar navigation, a search index, code highlighting, callout boxes, card layouts, and responsive design:

/data/projects/{project-name}/
  site/
    index.html
    *.html
    assets/
      style.css
      search.js
      theme-toggle.js
      highlight.js
    search-index.json

AI Provider System

docsfy uses a provider pattern to support multiple AI CLI tools through a unified interface:

@dataclass(frozen=True)
class ProviderConfig:
    binary: str
    build_cmd: Callable
    uses_own_cwd: bool = False

Each provider is invoked as a subprocess with prompts passed via stdin:

Provider Binary Command CWD Handling
Claude claude claude --model <model> --dangerously-skip-permissions -p subprocess cwd = repo path
Gemini gemini gemini --model <model> --yolo subprocess cwd = repo path
Cursor agent agent --force --model <model> --print --workspace <path> --workspace flag

AI responses are parsed using a multi-strategy JSON extraction pipeline — direct parse, brace-matching, markdown code block extraction, and regex recovery — to handle the variety of output formats that different AI tools produce.

Tip: Claude Code is the default provider. It uses the claude-opus-4-6[1m] model with 1 million token context, making it well-suited for analyzing large repositories in a single pass.

Quick Start

1. Configure Environment

Create a .env file with your AI provider credentials:

# AI Configuration
AI_PROVIDER=claude
AI_MODEL=claude-opus-4-6[1m]
AI_CLI_TIMEOUT=60

# Claude - Option 1: API Key
ANTHROPIC_API_KEY=your-key-here

# Claude - Option 2: Vertex AI
# CLAUDE_CODE_USE_VERTEX=1
# CLOUD_ML_REGION=us-central1
# ANTHROPIC_VERTEX_PROJECT_ID=your-project-id

# Logging
LOG_LEVEL=INFO

2. Run with Docker Compose

services:
  docsfy:
    build: .
    ports:
      - "8000:8000"
    env_file: .env
    volumes:
      - ./data:/data
      - ~/.config/gcloud:/home/appuser/.config/gcloud:ro
      - ./cursor:/home/appuser/.config/cursor
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

3. Generate Documentation

curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/your-org/your-repo"}'

4. Access Your Docs

Once generation completes, your documentation is available at:

  • Live: http://localhost:8000/docs/your-repo/
  • Download: GET /api/projects/your-repo/download returns a .tar.gz archive for self-hosting

API Overview

Method Path Description
POST /api/generate Start doc generation for a repo URL
GET /api/status List all projects and their generation status
GET /api/projects/{name} Get project details (last generated, commit SHA, pages)
DELETE /api/projects/{name} Remove a project and its generated docs
GET /api/projects/{name}/download Download site as .tar.gz archive
GET /docs/{project}/{path} Serve generated static HTML docs
GET /health Health check

Technology Stack

Component Technology
Web framework FastAPI + uvicorn
Templating Jinja2
Markdown Python markdown library
Database SQLite
Search lunr.js (client-side)
Code highlighting highlight.js
Build system hatchling
Package manager uv
Container Docker (python:3.12-slim, multi-stage)
Python 3.12+

Note: docsfy uses uv exclusively as its package manager — pip is not supported. All AI CLI tools are installed unpinned (always latest) in the Docker image to ensure compatibility with the newest model features.
