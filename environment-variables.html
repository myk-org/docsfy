Environment Variables

Environment Variables

Complete reference for all environment variables used to configure docsfy. These variables control AI provider selection, authentication, CLI behavior, and logging.

Quick Reference

Variable Required Default Description
AI_PROVIDER No claude AI CLI provider to use
AI_MODEL No claude-opus-4-6[1m] Model name passed to the AI CLI
AI_CLI_TIMEOUT No 60 Maximum time (minutes) for AI CLI execution
ANTHROPIC_API_KEY Conditional — API key for Claude direct access
CLAUDE_CODE_USE_VERTEX No — Enable Vertex AI authentication for Claude
CLOUD_ML_REGION Conditional — Google Cloud region for Vertex AI
ANTHROPIC_VERTEX_PROJECT_ID Conditional — Google Cloud project ID for Vertex AI
GEMINI_API_KEY Conditional — API key for Gemini CLI
CURSOR_API_KEY Conditional — API key for Cursor Agent
LOG_LEVEL No INFO Application logging verbosity

Configuration File

docsfy loads environment variables from a .env file at the project root via the docker-compose.yaml env_file directive:

services:
  docsfy:
    build: .
    ports:
      - "8000:8000"
    env_file: .env

Create your .env from the provided template:

cp .env.example .env

Warning: Never commit your .env file to version control. It contains secrets such as API keys.

AI Configuration

These variables control which AI CLI tool docsfy uses and how it behaves during documentation generation.

AI_PROVIDER

Selects the AI CLI provider used for both the planning and content generation stages of the pipeline.
Type String
Default claude
Allowed values claude, gemini, cursor

Each provider maps to a specific binary and command pattern:

Provider value Binary Command template
claude claude claude --model <AI_MODEL> --dangerously-skip-permissions -p
gemini gemini gemini --model <AI_MODEL> --yolo
cursor agent agent --force --model <AI_MODEL> --print --workspace <path>

The provider is configured internally using a frozen dataclass:

@dataclass(frozen=True)
class ProviderConfig:
    binary: str
    build_cmd: Callable
    uses_own_cwd: bool = False

Most providers operate with the subprocess cwd set to the cloned repository path. Cursor is the exception — it uses its own --workspace flag and sets uses_own_cwd=True.

# .env
AI_PROVIDER=claude

Note: docsfy performs an availability check before starting generation by sending a lightweight "Hi" prompt to verify the selected CLI tool is installed and responsive.

AI_MODEL

Specifies the model name passed to the AI CLI via its --model flag.
Type String
Default claude-opus-4-6[1m]

The model value is passed directly to whichever provider is selected. Make sure the model name is valid for your chosen provider.

# Use the default Claude model
AI_MODEL=claude-opus-4-6[1m]

# Or specify a Gemini model when using the Gemini provider
AI_PROVIDER=gemini
AI_MODEL=gemini-2.5-pro

Tip: The model value is interpolated directly into the CLI command. Refer to your provider's documentation for available model identifiers.

AI_CLI_TIMEOUT

Maximum time in minutes that docsfy will wait for a single AI CLI invocation to complete. This applies to each subprocess call during both the planning and content generation stages.
Type Integer
Default 60
Unit Minutes

AI CLI tools are invoked asynchronously via asyncio.to_thread(subprocess.run, ...). If a call exceeds this timeout, the subprocess is terminated.

# Allow up to 2 hours for large repositories
AI_CLI_TIMEOUT=120

Note: Large repositories with many files may require a higher timeout. If generation consistently fails with timeout errors, increase this value.

Authentication

Authentication variables are conditional — you must set the appropriate keys for whichever AI_PROVIDER you have selected. Each provider has its own authentication mechanism.

Claude Authentication

Claude Code supports two authentication methods. Use one of the following options.

Option 1: Direct API Key

ANTHROPIC_API_KEY=sk-ant-...
Variable ANTHROPIC_API_KEY
Type String
Required Yes, if using Claude without Vertex AI

Set your Anthropic API key for direct access to Claude.

Option 2: Google Vertex AI

For organizations using Claude through Google Cloud's Vertex AI platform, set all three of the following variables:

CLAUDE_CODE_USE_VERTEX=1
CLOUD_ML_REGION=us-east5
ANTHROPIC_VERTEX_PROJECT_ID=my-gcp-project-id
Variable Type Description
CLAUDE_CODE_USE_VERTEX Boolean (1 to enable) Switches Claude to use Vertex AI authentication
CLOUD_ML_REGION String Google Cloud region where your Vertex AI endpoint is deployed
ANTHROPIC_VERTEX_PROJECT_ID String Google Cloud project ID with Vertex AI access enabled

When using Vertex AI, your Google Cloud credentials must also be available inside the container. The docker-compose.yaml mounts them as a read-only volume:

volumes:
  - ~/.config/gcloud:/home/appuser/.config/gcloud:ro

Warning: When using Vertex AI, all three variables (CLAUDE_CODE_USE_VERTEX, CLOUD_ML_REGION, ANTHROPIC_VERTEX_PROJECT_ID) must be set together. Setting only some of them will result in authentication failures.

Gemini Authentication

GEMINI_API_KEY=AIza...
Variable GEMINI_API_KEY
Type String
Required Yes, if AI_PROVIDER=gemini

API key for the Google Gemini CLI. Additionally, Google Cloud credentials are mounted into the container for Gemini CLI access:

volumes:
  - ~/.config/gcloud:/home/appuser/.config/gcloud:ro

Cursor Authentication

CURSOR_API_KEY=cur-...
Variable CURSOR_API_KEY
Type String
Required Yes, if AI_PROVIDER=cursor

API key for the Cursor Agent CLI. Cursor configuration is mounted via a dedicated volume:

volumes:
  - ./cursor:/home/appuser/.config/cursor

Logging

LOG_LEVEL

Controls the verbosity of application logging output.
Type String
Default INFO
Allowed values DEBUG, INFO, WARNING, ERROR, CRITICAL 
# Default logging
LOG_LEVEL=INFO

# Verbose logging for debugging AI CLI interactions
LOG_LEVEL=DEBUG

# Quieter logging for production
LOG_LEVEL=WARNING

Tip: Set LOG_LEVEL=DEBUG when troubleshooting generation failures. This will include detailed output from AI CLI subprocess invocations.

Complete .env.example

Below is the full .env.example template with all supported variables:

# AI Configuration
AI_PROVIDER=claude
AI_MODEL=claude-opus-4-6[1m]
AI_CLI_TIMEOUT=60

# Claude - Option 1: API Key
# ANTHROPIC_API_KEY=

# Claude - Option 2: Vertex AI
# CLAUDE_CODE_USE_VERTEX=1
# CLOUD_ML_REGION=
# ANTHROPIC_VERTEX_PROJECT_ID=

# Gemini
# GEMINI_API_KEY=

# Cursor
# CURSOR_API_KEY=

# Logging
LOG_LEVEL=INFO

Docker and Container Considerations

docsfy runs as a non-root user (appuser) inside a python:3.12-slim container. Environment variables are injected via the env_file directive in docker-compose.yaml.

Volume Mounts for Credentials

Some authentication methods require file-based credentials in addition to environment variables:

Mount Purpose Mode
~/.config/gcloud:/home/appuser/.config/gcloud Google Cloud credentials for Vertex AI and Gemini Read-only
./cursor:/home/appuser/.config/cursor Cursor CLI configuration Read-write
./data:/data Persistent storage for generated docs and database Read-write

Note: The gcloud credentials mount is read-only (:ro) to prevent the container from modifying your local Google Cloud configuration.

Overriding at Runtime

You can override any environment variable at runtime without modifying your .env file:

# Override provider for a single run
docker compose run -e AI_PROVIDER=gemini -e GEMINI_API_KEY=your-key docsfy

# Override timeout for a large repository
docker compose run -e AI_CLI_TIMEOUT=120 docsfy
