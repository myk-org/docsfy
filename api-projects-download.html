GET /api/projects/{name}/download

GET /api/projects/{name}/download

Download the complete generated documentation site as a .tar.gz archive for self-hosting.

Overview

This endpoint packages the fully rendered static HTML documentation site for a project into a compressed .tar.gz archive. Use it to download your generated docs and host them on any static file server — Nginx, Apache, S3, GitHub Pages, Netlify, or any other platform that serves static files.

Endpoint

GET /api/projects/{name}/download

Path Parameters

Parameter Type Required Description
name string Yes The project name used during generation

Request

No request body or query parameters are required. The endpoint only needs the project name in the URL path.

Example Request

curl -O -J http://localhost:8000/api/projects/my-project/download

The -O flag saves the file using the server-provided filename, and -J uses the Content-Disposition header for the filename.

Saving to a Specific Path

curl -o my-project-docs.tar.gz http://localhost:8000/api/projects/my-project/download

Response

Success (200 OK)

Returns a binary .tar.gz archive containing the complete static documentation site.

Response Headers:

Header Value
Content-Type application/gzip
Content-Disposition attachment; filename="{name}-docs.tar.gz"

Error Responses

Status Code Description
404 Not Found Project does not exist
409 Conflict Project documentation is still generating (status is generating)
500 Internal Server Error Archive creation failed

Archive Contents

The downloaded archive mirrors the rendered site directory structure stored at /data/projects/{name}/site/:

{name}-docs/
├── index.html              # Landing page
├── getting-started.html    # Generated documentation pages
├── api-reference.html
├── *.html                  # Additional pages from plan.json
├── assets/
│   ├── style.css           # Theme styles (dark/light mode)
│   ├── search.js           # Client-side search functionality
│   ├── theme-toggle.js     # Dark/light theme switcher
│   └── highlight.js        # Code syntax highlighting
└── search-index.json       # Pre-built search index

The HTML pages include:

  • Sidebar navigation generated from the plan.json structure
  • Dark/light theme toggle with persistent preference
  • Client-side search powered by a pre-built search-index.json index
  • Code syntax highlighting via highlight.js
  • Responsive design for mobile and desktop
  • Callout boxes for notes, warnings, and info blocks

Usage Examples

Download and Extract

# Download the archive
curl -o docs.tar.gz http://localhost:8000/api/projects/my-project/download

# Extract the archive
tar -xzf docs.tar.gz

# View the site locally
cd my-project-docs
python -m http.server 3000

Self-Host with Nginx

Download the archive and configure Nginx to serve the static files:

# Download and extract to your web root
curl -s http://localhost:8000/api/projects/my-project/download | tar -xz -C /var/www/

server {
    listen 80;
    server_name docs.example.com;

    root /var/www/my-project-docs;
    index index.html;

    location / {
        try_files $uri $uri.html $uri/ =404;
    }

    # Cache static assets
    location /assets/ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}

Self-Host with Docker (Nginx)

FROM nginx:alpine
COPY my-project-docs/ /usr/share/nginx/html/

docker build -t my-docs .
docker run -p 8080:80 my-docs

Deploy to GitHub Pages

# Download and extract
curl -o docs.tar.gz http://localhost:8000/api/projects/my-project/download
tar -xzf docs.tar.gz

# Push to GitHub Pages
cd my-project-docs
git init
git checkout -b gh-pages
git add .
git commit -m "Deploy documentation"
git remote add origin git@github.com:your-org/your-repo.git
git push -f origin gh-pages

Deploy to AWS S3

# Download and extract
curl -o docs.tar.gz http://localhost:8000/api/projects/my-project/download
tar -xzf docs.tar.gz

# Sync to S3 bucket
aws s3 sync my-project-docs/ s3://my-docs-bucket/ --delete

# Set content types for assets
aws s3 cp s3://my-docs-bucket/assets/ s3://my-docs-bucket/assets/ \
  --recursive --metadata-directive REPLACE \
  --cache-control "max-age=604800"

Scripted Download with Status Check

Before downloading, verify the project has finished generating:

#!/bin/bash
PROJECT="my-project"
BASE_URL="http://localhost:8000"

# Check project status
STATUS=$(curl -s "${BASE_URL}/api/projects/${PROJECT}" | python -m json.tool | grep -o '"status": "[^"]*"')

if echo "$STATUS" | grep -q "ready"; then
    echo "Downloading documentation..."
    curl -o "${PROJECT}-docs.tar.gz" "${BASE_URL}/api/projects/${PROJECT}/download"
    echo "Downloaded ${PROJECT}-docs.tar.gz"
else
    echo "Project is not ready. Current status: ${STATUS}"
    exit 1
fi

Python Client Example

import httpx
import tarfile
from pathlib import Path


def download_docs(base_url: str, project_name: str, output_dir: Path) -> None:
    """Download and extract project documentation."""
    # Check project status first
    status_resp = httpx.get(f"{base_url}/api/projects/{project_name}")
    status_resp.raise_for_status()

    project = status_resp.json()
    if project["status"] != "ready":
        raise RuntimeError(
            f"Project '{project_name}' is not ready: {project['status']}"
        )

    # Download the archive
    download_url = f"{base_url}/api/projects/{project_name}/download"
    archive_path = output_dir / f"{project_name}-docs.tar.gz"

    with httpx.stream("GET", download_url) as response:
        response.raise_for_status()
        with open(archive_path, "wb") as f:
            for chunk in response.iter_bytes():
                f.write(chunk)

    # Extract
    with tarfile.open(archive_path, "r:gz") as tar:
        tar.extractall(path=output_dir)

    print(f"Documentation extracted to {output_dir / f'{project_name}-docs'}")


download_docs("http://localhost:8000", "my-project", Path("./output"))

Workflow

The download endpoint fits into the docsfy generation lifecycle as follows:

  1. Generate — POST /api/generate with the repository URL to start documentation generation
  2. Monitor — GET /api/projects/{name} to check status until it reports ready
  3. Preview — GET /docs/{name}/ to preview the hosted documentation directly from docsfy
  4. Download — GET /api/projects/{name}/download to get the archive for self-hosting
# Full workflow example
PROJECT="my-project"

# 1. Start generation
curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/your-org/your-repo"}'

# 2. Poll for completion
while true; do
  STATUS=$(curl -s http://localhost:8000/api/projects/${PROJECT} | python -c "import sys,json; print(json.load(sys.stdin)['status'])")
  echo "Status: $STATUS"
  [ "$STATUS" = "ready" ] && break
  sleep 10
done

# 3. Download the site
curl -o ${PROJECT}-docs.tar.gz http://localhost:8000/api/projects/${PROJECT}/download
echo "Documentation downloaded: ${PROJECT}-docs.tar.gz"

Note: The project name is derived from the repository URL during generation. For example, https://github.com/your-org/my-project produces the project name my-project.

Storage Reference

The download endpoint packages files from the project's site/ directory on the server filesystem:

/data/projects/{project-name}/
  plan.json             # Documentation structure (not included in archive)
  cache/
    pages/*.md          # AI-generated markdown (not included in archive)
  site/                 # This directory is archived and returned
    index.html
    *.html
    assets/
      style.css
      search.js
      theme-toggle.js
      highlight.js
    search-index.json

Tip: Only the site/ directory is included in the download. The plan.json and cache/ directory are internal to docsfy and used for incremental updates when documentation is regenerated.

Warning: If you re-generate documentation for a project while a download is in progress, the archive contents may be inconsistent. Wait for generation to complete (status ready) before downloading.
