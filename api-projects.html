GET /api/projects/{name}

GET /api/projects/{name}

Retrieve detailed information about a specific documentation project, including its current status, the last generated timestamp, the commit SHA used for generation, and the full list of generated pages.

Endpoint

GET /api/projects/{name}

Path Parameters

Parameter Type Required Description
name string Yes The project name. This matches the directory name under /data/projects/.

Response

200 OK

Returns the full project details.

{
  "name": "docsfy",
  "repo_url": "https://github.com/username/docsfy.git",
  "status": "ready",
  "last_generated": "2026-03-04T14:32:10Z",
  "last_commit_sha": "c6db255a9f1e4b8d7c3a2e6f0b1d5c8a4e7f9b2d",
  "pages": [
    {
      "title": "Introduction",
      "path": "index.html"
    },
    {
      "title": "Getting Started",
      "path": "getting-started.html"
    },
    {
      "title": "API Reference",
      "path": "api-reference.html"
    },
    {
      "title": "Configuration",
      "path": "configuration.html"
    }
  ]
}

Response Fields

Field Type Description
name string The project name.
repo_url string The source GitHub repository URL (SSH or HTTPS).
status string Current project status. One of generating, ready, or error.
last_generated string ISO 8601 timestamp of the most recent successful generation. null if the project has never completed generation.
last_commit_sha string The full Git commit SHA that was used for the last generation. Used internally for incremental updates. null if no generation has completed.
pages array List of pages in the generated documentation site, derived from the project's plan.json. Empty array if generation is still in progress.

Project Status Values

Status Description
generating Documentation generation is currently in progress. The pipeline is running through clone, planning, content generation, or HTML rendering stages.
ready Documentation has been successfully generated and is available for serving or download.
error The most recent generation attempt failed. Check generation logs for details.

404 Not Found

Returned when no project with the given name exists.

{
  "detail": "Project not found: my-project"
}

curl http://localhost:8000/api/projects/broken-repo

{
  "name": "broken-repo",
  "repo_url": "https://github.com/org/broken-repo.git",
  "status": "error",
  "last_generated": "2026-03-03T09:15:00Z",

      "title": "Introduction",
      "path": "index.html"
    }
  ]
}

Note: When a project has status: "error", the last_generated and pages fields reflect the last successful generation, not the failed attempt. If no generation has ever succeeded, last_generated will be null and pages will be empty.

Incremental Updates

The last_commit_sha field plays a key role in docsfy's incremental update strategy. When a project is re-generated via POST /api/generate, docsfy compares the current repository HEAD against the stored last_commit_sha to determine what has changed:

  1. If the SHA matches, no regeneration is needed
  2. If the SHA differs, the AI Planner re-evaluates the documentation structure
  3. Only pages affected by the changes are regenerated, keeping cached pages intact

This means a project's last_commit_sha always reflects the repository state from which the current documentation was built.

Tip: Poll this endpoint after triggering a generation with POST /api/generate to track when the status transitions from generating to ready or error.

Storage Details

The data returned by this endpoint is assembled from two sources:

  • SQLite database (/data/docsfy.db) — stores the project metadata: name, repo URL, status, last generated timestamp, last commit SHA, and generation history
  • Filesystem (/data/projects/{name}/plan.json) — the AI-generated documentation plan that defines the page list, sections, and navigation hierarchy

The filesystem structure for a project looks like:

/data/projects/{project-name}/
  plan.json             # Doc structure from AI (pages, sections, navigation)
  cache/
    pages/*.md          # AI-generated markdown (cached for incremental updates)
  site/                 # Final rendered static HTML
    index.html
    *.html
    assets/
      style.css
      search.js
      theme-toggle.js
      highlight.js
    search-index.json

Related Endpoints

Method Path Description
POST /api/generate Start documentation generation for a repository
GET /api/status List all projects and their generation status
DELETE /api/projects/{name} Remove a project and its generated docs
GET /api/projects/{name}/download Download the generated site as a .tar.gz archive
GET /docs/{project}/{path} Serve the generated static HTML documentation

Warning: Calling DELETE /api/projects/{name} will permanently remove all generated documentation, cached markdown, and metadata for the project. This action cannot be undone.
