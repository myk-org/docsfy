GET /docs/{project}/{path}

GET /docs/{project}/{path}

Serve generated static HTML documentation pages directly from the docsfy server.

Overview

The GET /docs/{project}/{path} endpoint serves the final rendered HTML documentation for a project. After docsfy's generation pipeline produces a static site, this endpoint makes it immediately browsable — no separate web server or download step required.

Each project's documentation lives under its own URL namespace, making it easy to host multiple projects from a single docsfy instance.

URL Parameters

Parameter Type Description
project string The project name (derived from the repository name during generation)
path string The path to a specific file within the generated site (e.g., index.html, getting-started.html, assets/style.css)

How It Works

When a request hits /docs/{project}/{path}, docsfy resolves the path against the project's generated site directory on the filesystem:

/data/projects/{project}/site/{path}

This directory is populated by Stage 4 (HTML Renderer) of the generation pipeline. The endpoint serves whatever file matches the requested path — HTML pages, CSS stylesheets, JavaScript files, search indexes, and any other static assets.

Request Flow

Client Request
    GET /docs/my-project/getting-started.html
         │
         ▼
    FastAPI Router
         │
         ▼
    Resolve filesystem path
    /data/projects/my-project/site/getting-started.html
         │
         ▼
    Return file with appropriate Content-Type

Usage Examples

Browse a Project's Documentation

Navigate to the root page of a project's documentation:

GET /docs/my-project/index.html

Access a Specific Page

Each page defined in the project's plan.json becomes an HTML file in the site directory:

GET /docs/my-project/getting-started.html
GET /docs/my-project/api-reference.html
GET /docs/my-project/configuration.html

Load Static Assets

The generated site includes bundled CSS, JavaScript, and other assets under the assets/ directory:

GET /docs/my-project/assets/style.css
GET /docs/my-project/assets/search.js
GET /docs/my-project/assets/theme-toggle.js
GET /docs/my-project/assets/highlight.js

Fetch the Search Index

Client-side search is powered by a JSON index that is also served through this endpoint:

GET /docs/my-project/search-index.json

Generated Site Structure

Every project's site directory follows a consistent layout:

/data/projects/{project}/site/
├── index.html              # Landing page
├── getting-started.html    # Example documentation page
├── api-reference.html      # Example documentation page
├── ...                     # Additional pages from plan.json
├── assets/
│   ├── style.css           # Theme styles (dark/light mode)
│   ├── search.js           # Client-side search (lunr.js)
│   ├── theme-toggle.js     # Dark/light theme switcher
│   └── highlight.js        # Code syntax highlighting
└── search-index.json       # Search index for client-side search

Note: The exact set of HTML pages varies per project. The AI Planner (Stage 2) determines the documentation structure, and the AI Content Generator (Stage 3) produces the content for each page. The HTML Renderer (Stage 4) then converts everything into the final static site.

Site Features

Pages served through this endpoint include the full set of docsfy rendering features:

Feature Description
Sidebar navigation Hierarchical page navigation generated from plan.json
Dark/light theme Toggle between themes; preference persisted client-side
Client-side search Full-text search across all pages using lunr.js
Syntax highlighting Code blocks highlighted with highlight.js
Responsive design Adapts to mobile, tablet, and desktop viewports
Card layouts Visual grouping for related content
Callout boxes Styled note, warning, and info blocks

Relationship to the Generation Pipeline

This endpoint only serves documentation that has completed the full generation pipeline. The pipeline stages are:

  1. Clone Repository — Shallow clone of the source repo
  2. AI Planner — Analyzes the repo and produces plan.json
  3. AI Content Generator — Writes markdown for each page
  4. HTML Renderer — Converts markdown + plan into the static site served by this endpoint

Warning: If a project's status is generating or error, the documentation may not be available or may be incomplete. Check the project status via GET /api/projects/{name} before linking to documentation pages.

Project Status and Availability

Before serving documentation, verify the project has completed generation:

# Check project status
curl http://localhost:8000/api/projects/my-project
Status Docs available? Description
ready Yes Generation completed successfully; docs are served
generating Partial/No Pipeline is still running
error No Generation failed; check logs for details

Error Responses

Scenario Expected Behavior
Project does not exist 404 Not Found
Path does not exist within a project's site 404 Not Found
Project is in generating state Docs may not yet be available
Project is in error state Docs are unavailable

Self-Hosting Alternative

If you prefer to serve documentation from your own infrastructure instead of through docsfy's built-in endpoint, download the complete site as a .tar.gz archive:

# Download the static site for self-hosting
curl -O http://localhost:8000/api/projects/my-project/download

The archive contains the same files served by GET /docs/{project}/{path}, ready to deploy to any static file host (Nginx, Apache, S3, GitHub Pages, etc.).

Tip: Use GET /docs/{project}/{path} during development and testing for instant access. For production, consider downloading the static site and hosting it behind a CDN for optimal performance.

Storage Configuration

The documentation files are stored in the persistent data volume. In the default Docker Compose configuration:

services:
  docsfy:
    volumes:
      - ./data:/data    # All project data including generated sites

This maps the container's /data directory — where all project sites live — to ./data on the host. The generated sites persist across container restarts.

Note: Ensure the data volume has sufficient disk space. Each project's generated site includes HTML pages, CSS/JS assets, and a search index. Size varies based on the complexity of the documented repository.
