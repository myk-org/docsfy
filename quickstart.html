Quickstart

Quickstart

Get docsfy running in under 5 minutes with Docker Compose, generate your first documentation site, and view the results.

Prerequisites

Before you begin, make sure you have the following installed:

  • Docker (v20.10+) and Docker Compose (v2.0+)
  • An API key for your chosen AI provider:
  • Claude — an ANTHROPIC_API_KEY or Google Cloud credentials for Vertex AI
  • Gemini — a GEMINI_API_KEY
  • Cursor — a CURSOR_API_KEY

Tip: Claude is the default AI provider. If you're unsure which to pick, start with Claude.

Step 1: Clone the Repository

git clone https://github.com/your-org/docsfy.git
cd docsfy

Step 2: Configure Environment Variables

Copy the example environment file and fill in your API key:

cp .env.example .env

Open .env in your editor. At minimum, set your AI provider credentials:

# AI Configuration
AI_PROVIDER=claude
AI_MODEL=claude-opus-4-6[1m]
AI_CLI_TIMEOUT=60

# Claude - Option 1: API Key
ANTHROPIC_API_KEY=sk-ant-...

# Logging
LOG_LEVEL=INFO

Note: You only need to configure credentials for the provider you plan to use. Leave the other provider keys commented out.

Alternative: Using Vertex AI for Claude

If you're running Claude through Google Cloud Vertex AI instead of a direct API key, configure these variables instead:

AI_PROVIDER=claude
CLAUDE_CODE_USE_VERTEX=1
CLOUD_ML_REGION=us-east5
ANTHROPIC_VERTEX_PROJECT_ID=your-gcp-project-id

When using Vertex AI, you also need to mount your gcloud credentials into the container (already configured in the default docker-compose.yaml).

Step 3: Start docsfy

Launch the service with Docker Compose:

docker compose up -d

This builds the container image (first run only) and starts the docsfy service. The container:

  • Runs on python:3.12-slim with all AI CLI tools pre-installed
  • Exposes the API on port 8000
  • Persists generated documentation in a ./data volume

Verify the service is healthy:

curl http://localhost:8000/health

You should see a successful health check response.

Tip: Use docker compose logs -f docsfy to follow the service logs during generation.

Step 4: Generate Your First Documentation Site

Trigger documentation generation by sending a POST request with a GitHub repository URL:

curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/expressjs/express"}'

docsfy runs a four-stage pipeline for every generation request:

  1. Clone — Shallow clone (--depth 1) of the repository
  2. AI Planner — The AI analyzes the codebase and produces a plan.json defining the documentation structure (pages, sections, navigation)
  3. AI Content Generator — For each page in the plan, the AI explores the source code and writes markdown content (pages run concurrently for speed)
  4. HTML Renderer — Markdown is converted to polished static HTML with Jinja2 templates, sidebar navigation, dark/light theme, search, and syntax highlighting

Note: Generation time depends on the size of the repository and your AI provider. A typical project takes a few minutes. The AI_CLI_TIMEOUT variable (default: 60 minutes) controls the maximum time allowed.

Step 5: Check Generation Status

Monitor progress by querying the status endpoint:

curl http://localhost:8000/api/status

This returns a list of all projects with their current status: generating, ready, or error.

To get details for a specific project:

curl http://localhost:8000/api/projects/express

This returns project metadata including the last generated timestamp, commit SHA, and page list.

Step 6: View Your Documentation

Once the status shows ready, open your browser and navigate to:

http://localhost:8000/docs/express/

You'll see a fully rendered documentation site with:

  • Sidebar navigation generated from the AI's documentation plan
  • Dark/light theme toggle for comfortable reading
  • Client-side search across all pages
  • Syntax-highlighted code blocks for code examples
  • Responsive design that works on any screen size

Downloading for Self-Hosting

If you want to host the generated documentation on your own infrastructure (GitHub Pages, Netlify, S3, etc.), download the static site as a .tar.gz archive:

curl -O http://localhost:8000/api/projects/express/download

The archive contains the complete static site from /data/projects/express/site/ — just extract and deploy:

tar -xzf express.tar.gz
# Upload the contents to your hosting provider

Docker Compose Reference

Here is the full docker-compose.yaml for reference:

services:
  docsfy:
    build: .
    ports:
      - "8000:8000"
    env_file: .env
    volumes:
      - ./data:/data
      - ~/.config/gcloud:/home/appuser/.config/gcloud:ro
      - ./cursor:/home/appuser/.config/cursor
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
Volume Mount Purpose
./data:/data Persists the SQLite database (docsfy.db) and all generated documentation
~/.config/gcloud:ro Passes Google Cloud credentials for Vertex AI (read-only)
./cursor Cursor agent configuration directory

Warning: The ./data volume contains all generated sites and project metadata. Do not delete it unless you want to start fresh.

Regenerating Documentation

When a repository is updated, you can regenerate its documentation by calling the same endpoint:

curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/expressjs/express"}'

docsfy supports incremental updates — it tracks the last commit SHA per project and only regenerates pages affected by changes, making subsequent runs faster than the initial generation.

Removing a Project

To remove a project and all its generated documentation:

curl -X DELETE http://localhost:8000/api/projects/express
