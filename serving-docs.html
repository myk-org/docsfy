Serving Documentation

Serving Documentation

docsfy provides two ways to access your generated documentation: serve it directly from the docsfy API, or download it as a .tar.gz archive for self-hosting on your own infrastructure.

Serving Directly from docsfy

Once documentation has been generated for a project, docsfy serves the rendered static HTML site automatically through its built-in FastAPI server.

Accessing Served Documentation

Generated documentation is available at:

GET /docs/{project}/{path}

where {project} is the project name and {path} is the page path within the generated site. For example, if you generated docs for a project named my-app:

http://localhost:8000/docs/my-app/
http://localhost:8000/docs/my-app/index.html
http://localhost:8000/docs/my-app/getting-started.html

The served site includes all rendered assets — CSS, JavaScript, search index, and syntax highlighting — so no additional setup is required.

Starting the Server

docsfy runs as a FastAPI application served by uvicorn. The default entrypoint starts the server on all interfaces at port 8000:

uv run --no-sync uvicorn docsfy.main:app --host 0.0.0.0 --port 8000

With Docker Compose, this is handled automatically:

services:
  docsfy:
    build: .
    ports:
      - "8000:8000"
    env_file: .env
    volumes:
      - ./data:/data
      - ~/.config/gcloud:/home/appuser/.config/gcloud:ro
      - ./cursor:/home/appuser/.config/cursor
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

Tip: Use the GET /health endpoint to verify the server is running and ready to serve requests.

Verifying Project Status

Before accessing served documentation, confirm the project has finished generating:

curl http://localhost:8000/api/projects/my-app

The response includes the project status. Documentation is only servable when the status is ready:

Status Meaning
generating Documentation generation is in progress
ready Generation complete; docs are being served
error Generation failed; check logs for details

You can also list all projects and their statuses:

curl http://localhost:8000/api/status

Note: Attempting to access /docs/{project}/ for a project that is still generating or in an error state will not return the expected documentation.

What Gets Served

The static site served from /docs/{project}/ is rendered from the contents of /data/projects/{project-name}/site/ on the filesystem. This includes:

/data/projects/{project-name}/site/
  index.html
  *.html
  assets/
    style.css
    search.js
    theme-toggle.js
    highlight.js
  search-index.json

The served pages include:

  • Sidebar navigation generated from plan.json
  • Dark/light theme toggle with persistent preference
  • Client-side search powered by lunr.js
  • Code syntax highlighting via highlight.js
  • Responsive design for mobile and desktop
  • Callout boxes for notes, warnings, and informational content

Downloading as a tar.gz Archive

For self-hosting scenarios — deploying to your own web server, CDN, or static hosting platform — docsfy provides a download endpoint that packages the entire generated site into a .tar.gz archive.

Download Endpoint

GET /api/projects/{name}/download

Download the archive using curl:

curl -O http://localhost:8000/api/projects/my-app/download

Or specify an output filename:

curl -o my-app-docs.tar.gz http://localhost:8000/api/projects/my-app/download

Warning: The download endpoint is only available for projects with a ready status. Ensure generation has completed before attempting to download.

Archive Contents

The downloaded .tar.gz contains the complete static site — the same files served by the /docs/{project}/ route:

site/
├── index.html
├── *.html              (individual documentation pages)
├── assets/
│   ├── style.css       (bundled CSS styles)
│   ├── search.js       (client-side search functionality)
│   ├── theme-toggle.js (dark/light mode toggle)
│   └── highlight.js    (code syntax highlighting)
└── search-index.json   (pre-built search index)

The archive is fully self-contained. All CSS, JavaScript, and search index data are bundled — there are no external CDN dependencies.

Self-Hosting the Archive

After downloading, extract the archive and serve the contents with any static file server.

Extract the archive:

tar -xzf my-app-docs.tar.gz

Serve with Python's built-in HTTP server:

cd site/
python3 -m http.server 3000

Serve with nginx:

server {
    listen 80;
    server_name docs.example.com;

    root /var/www/my-app-docs/site;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}

Deploy to static hosting platforms:

The extracted site/ directory can be deployed directly to platforms such as GitHub Pages, Netlify, Vercel, Cloudflare Pages, or Amazon S3 with CloudFront.

Tip: Since the site uses client-side search with a pre-built search-index.json, search functionality works without any server-side component — making it ideal for static hosting.

Persistent Storage Configuration

Both serving and downloading depend on the /data volume where docsfy stores generated sites. When running with Docker Compose, ensure the volume is mounted:

volumes:
  - ./data:/data

This volume contains:

Path Purpose
/data/docsfy.db SQLite database with project metadata and status
/data/projects/{name}/plan.json Documentation structure from AI planner
/data/projects/{name}/cache/pages/*.md Cached AI-generated markdown (used for incremental updates)
/data/projects/{name}/site/ Final rendered HTML site (served and downloadable)

Warning: Deleting the /data volume will remove all generated documentation sites. Back up this directory if your generated docs are important.

Managing Served Projects

Removing a Project

To stop serving a project and delete its generated documentation:

curl -X DELETE http://localhost:8000/api/projects/my-app

This removes the project entry from the database and deletes all associated files from /data/projects/my-app/.

Regenerating Documentation

To update documentation after repository changes, submit a new generation request for the same repository:

curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/org/my-app"}'

docsfy supports incremental updates — it tracks the last commit SHA per project and only regenerates pages affected by changes. Once regeneration completes, the served documentation at /docs/{project}/ reflects the updated content automatically, and a fresh archive is available from the download endpoint.
