DELETE /api/projects/{name}

Remove a project and all its generated documentation from the server. This permanently deletes the project metadata from the SQLite database and removes all associated files from the filesystem, including cached markdown, rendered HTML, and the documentation plan.

Endpoint

DELETE /api/projects/{name}

Path Parameters

Parameter Type Required Description
name string Yes The unique name of the project to delete. This is the project name assigned during generation via POST /api/generate.

Response

200 OK

The project and all associated documentation were successfully deleted.

{
  "status": "deleted",
  "project": "my-project"
}

404 Not Found

The specified project does not exist.

{
  "detail": "Project 'my-project' not found"
}

Examples

cURL

curl -X DELETE http://localhost:8000/api/projects/my-project

Python (requests)

import requests

response = requests.delete("http://localhost:8000/api/projects/my-project")

if response.status_code == 200:
    print(f"Project deleted: {response.json()}")
elif response.status_code == 404:
    print(f"Project not found: {response.json()['detail']}")

Python (httpx, async)

import httpx

async with httpx.AsyncClient() as client:
    response = await client.delete("http://localhost:8000/api/projects/my-project")
    response.raise_for_status()
    print(response.json())

What Gets Deleted

When a project is deleted, the following resources are permanently removed:

Database Records

Project metadata is deleted from the SQLite database at /data/docsfy.db, including:

  • Project name and repository URL
  • Generation status (generating / ready / error)
  • Last generated timestamp and commit SHA
  • Generation history and logs

Filesystem

The entire project directory at /data/projects/{name}/ is removed, which includes:

/data/projects/{name}/
  plan.json               # documentation structure from AI planner
  cache/
    pages/*.md            # AI-generated markdown content
  site/                   # rendered static HTML site
    index.html
    *.html
    assets/
      style.css
      search.js
      theme-toggle.js
      highlight.js
    search-index.json

Warning: This operation is irreversible. All generated documentation, cached markdown pages, and the AI-produced documentation plan will be permanently lost. To regenerate, you must submit a new POST /api/generate request, which will re-clone the repository and run the full AI generation pipeline from scratch.

Behavior During Active Generation

If a project is currently in the generating state, the delete request will remove the project record and any partially written files. Any in-progress AI generation tasks for the project will be cancelled.

Note: Check the project status with GET /api/projects/{name} before deleting to confirm whether generation is still in progress.

# Check project status before deleting
curl http://localhost:8000/api/projects/my-project

# Then delete
curl -X DELETE http://localhost:8000/api/projects/my-project

Served Documentation

Once a project is deleted, its documentation is immediately no longer accessible at the serving endpoint:

GET /docs/{name}/{path}

Any requests to the project's documentation pages will return a 404 Not Found after deletion.

Tip: If you need to preserve a copy of the generated documentation before deleting, download the site archive first using GET /api/projects/{name}/download, which provides the complete static HTML site as a .tar.gz file suitable for self-hosting.

# Download a backup before deleting
curl -o my-project-docs.tar.gz http://localhost:8000/api/projects/my-project/download

# Then safely delete
curl -X DELETE http://localhost:8000/api/projects/my-project

Re-generating After Deletion

After deleting a project, you can regenerate its documentation by submitting a new generation request:

curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/owner/repo"}'

Note: Unlike incremental updates (which compare commit SHAs and regenerate only affected pages), a full generation after deletion runs the complete four-stage pipeline: repository cloning, AI planning, AI content generation, and HTML rendering.
